Rendering and redirection

Pros and cons of the different approaches.

Both rendering and redirection can be performed either client side or server side.


Rendering client side

Client side rendering (CSR) is ideal for dynamic websites. It may be that the first page is slow to load but then the browser can update content very quickly, maybe using linked JavaScript files, without having to wait for a series of requests to processed and returned by the server. This has the advantage of making it easier to provide a high UX, but a disadvantage that the site may not perform well with SEO (Search Engine Optimisation) because the web crawlers cannot read all the relevant content. An endpoint in our Express server fight look like this: 

            
            app.get("/", (req, res) => {
                res.sendFile(__dirname + "/public/frontpage.html")
            });


Rendering server side

Server side rendering (SSR) is good for static websites. After receiving a request for a page, the server accesses all data required, renders the page to match the request, and returns a complete HTML file. The browser only needs to execute the HTML, which is fast, and the site scores well in SEO as all data is available. But if the page needs updating, for example after user input, then the process is slow and server load high. An endpoint in our Express server fight look like this: 

            
            const frontpage = templateEngine.readPage('./public/pages/frontpage/frontpage.html');
            const frontpagePage = templateEngine.renderPage(frontpage, {
            tabTitle: 'NJS: Home'
            });

            app.get('/', ((req, res) => {
                res.send(frontpagePage);
            }));


Redirection

Redirection is sending the browser to an alternative url. Here are four ways to do it:

Client side 

            
            window.location.href = 'https://example.com/new-page';

Server side 

            
            // With Express
            res.redirect(301, 'https://example.com/new-page');

Middleware 

            
            // With Express's own
            app.use((req, res, next) => {
                if (shouldRedirect(req)) {
                    res.redirect(301, 'https://example.com/new-page');
                } else {
                    next();
                }
            });

Proxy

Use a reverse proxy like Nginx or Apache to handle redirects before requests even reach your Node.js application.
