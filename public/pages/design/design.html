Design

Organising the structure of the code

 The aim is to produce clean code which promotes readability, reusability, professionalism, and error catching. To help with this we structure our code after accepted principles, and use tools to improve our formatting.


Server structure:

 Place imports at the top of the file, then HTML pages, then APIs ordred by HTTP method (GET, POST, PUT, PATCH, DELETE as described in the 'restapi' page) and finally the server code.

Assets:

 Create an 'assets' folder under 'public' to store static files like images, fonts, stylesheets, and JavaScript files. This helps keep your project organized and makes it easier to locate and manage assets. Seperate types of assets should be in their own subfolders, which can also be subdivided after functionality if required. 


/public
    /assets
        /css
        /images
        /js

 HTML files are grouped into a 'pages' directory. Each HTML file should represent a distinct page. We have also provided subfolders for categories of views.

Clean code:

 Writing clean code is esential, and there are several tools and techniques that can be used to help.

  • 'use strict' from ECMAScript 5 is, according to Mozilla, the way to opt out of 'sloppy mode'. It also helps to write cleaner code and catch errors by enforcing stricter parsing and error handling on the code at runtime. Not used here as modules are in strict mode by default.

  • Integrating linters and code formatters (e.g., ESLint and Prettier) into your development workflow enforces code consistency and catches errors early. We use ESLint with air b'n'b style.

  • Naming conventions: Filenames should be lowercase and no spaces, as different browsers/servers/languages handle them differently which can cause errors.

  • Focus on accessabilty: We can use semantic HTML tags (for example <section> in this html ) to make it easier to identify type of content. There is a whole standard called WCAG (Web Content Accessibility Guidelines) which helps screen readers and assistive technologies to interpret and navigate the content.
