Variables and data types

To be able to operate on variables, it is important to know something about the type.

 We will look at data types and type coercion in JavaScript. 'var' can be used to specify the data type, but we try to minimise its use as it has global scope which can be difficult to debug due to hoisting.
 Scope is an important concept. Opening a scope pushes a call execution stack and local variables which are then popped outside the scope. Block scoping gives more granular control of scope than function scoping.
 We use 'const' as much as possible, which does need to be assigned a value when declared and otherwise 'let' for local variables with changing values. A const cannot be redeclared, but an array can be changed, for example by pushing an element, and an object's properties can be changed.


Data types:

There are 8 data types in javascript

  • String - represents sequence of characters
  • Number - represents numeric values
  • Bigint - represents integers with arbitrary magnitude
  • Boolean - represents boolean value
  • Undefined - represents undefined value
  • Null - represents no value
  • Symbol - is a unique and immutable primitive value and may be used as the key of an Object property
  • Object (object, array, date) - the only mutable values. Functions are also objects

Although they seem similar, undefined indicates the absence of a value, while null indicates the absence of an object . JavaScript usually defaults to undefined when something is devoid of a value. You can find type using typeof(). 


const value = true;
console.log(typeof(value));

Dynamic types:

JavaScript is a dynamic language with dynamic types. Variables in JavaScript are not directly associated with any particular value type and any variable can be assigned (and re-assigned) values of all types. 


let myVariable = 7; // myVariable is now a number
myVariable = "a prime number"; // myVariable is now a string
myVariable = true; // myVariable is now a boolean

Weak typing and type coercion:

It is also a weakly typed language. This means that you can often use a value of one type where another type is expected, and the language will convert it to the right type for you, instead of throwing type errors. This is type coercion. 


const myVariable = 7; // a number
const result = myVariable + "1"; // JavaScript coerces myVariable to a string
console.log(result); // "71"

When comparing data the == operator performs a loose check with type coercion, while === requires identical type.

JavaScript defines coercion rules:

  • Primitive coercion - where a primitive value is expected, but no strong preference for the actual type;
  • Numeric coercion - bigint, number;
  • Other coercions - string, boolean, object.

The details of coercion rules are beyond the scope of this document.
More info can be found on the MDN site at https://developer.mozilla.org/en-US/docs/Web/JavaScript/Data_structures
