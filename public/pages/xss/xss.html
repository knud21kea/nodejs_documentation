Bonus documentation covering basics of XSS

 Cross site scripting: What is is, how it works, how to combat it.


What is it?

XSS attacks are conducted by attackers who use vulnerable web applications to inject malicious code.

Return to top of page

How does it work?

The malicious code executes in the browser, the attacker steals cookies from account users and gains access to sensitive and confidential information in the application.

Return to top of page

How to combat an XXS attack?

There are basically two strategies, both of which need to be implemented.

  • Input validation is performed to make sure only secure data enters an information system
  • Output encoding translates untrusted input into a safe model, which can be done by sanitizing HTML.
    • Client-side sanitization: prevents unsafe content from the DOM level
    • Server-side sanitization: prevents the storage of malicious HTML content in databases
